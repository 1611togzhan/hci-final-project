# AI Usage

AI was used as a support tool during the development of this project.

## Where AI was used
- Understanding assignment requirements and breaking them into steps
- Generating initial Streamlit app structure and code
- Designing data cleaning and visualization features
- Debugging errors during development
- Structuring project files and documentation

## Dataset creation
A synthetic dataset was generated manually using Python in VS Code.  
This was done to ensure:
- more than 1000 rows (requirement)
- multiple data types (numeric, categorical, datetime)
- presence of missing values
- better testing of cleaning and transformation features

The dataset generation process was implemented independently and executed locally.

## What was verified manually
All functionality of the application was tested manually, including:
- File upload (CSV, XLSX, JSON)
- Data overview and profiling
- Missing value handling
- Duplicate removal
- Data type conversion
- Categorical cleaning and mapping
- Outlier handling
- Scaling and normalization
- Column operations
- Validation rules
- Visualization outputs
- Export functions

## Manual validation
All outputs were checked manually:
- cleaned datasets
- transformation reports
- visualization correctness
- error handling and edge cases

## Important note
AI was used only as a support tool.  
The final logic, testing, debugging, and validation of the application were performed manually.
